Deploy to Cloud Functions, Schedule invocations Using Cloud Scheduler

Nov 15, 2019 Cloud Computing GCP Cloud Scheduler Cloud Functions
Share on:

TL;DR: This post is about Scheduling Firebase Cloud Functions with Cloud Scheduler. Precisely we use Cloud Functions to deploy a birthday reminder app and use Cloud Scheduler to invoke the app.

A couple of weeks ago, I stumbled upon a repo on GitHub, it's a Flask app that uses Twilio to send SMS notifications about your friends' upcoming birthdays. It has a script that can be run as a background job using schedule, a lighweight python job scheduing library, to check once a day for birthdays and it sends out an SMS to your configured phone number if there is a birthday that day.

It's an interesting idea especially if you want to keep up with your friends' birthdays and don't want to use Facebook for it. Unless you can (or want to?) memorize all of them or laboriously save them in your contacts, you probably need to have a mechanism to help you remember them. Know that you may memorize them and still forget to remember them when the actual days come.

Facebook already has all of your friends' birthdays data, at least of those who decided to share them. How to export them is covered here.

This birthday app has a front-end with a page that displays birthdays by month, and other pages. But let's assumed we just have a backend that checks for birthdays and sends out SMS notifications, there is no front-end, at least not a 'meaningful' one.

The question we then have to ask ourselves is where should we deploy and run our minimal birthday reminder app? We can run it on our laptop; but what if you don't have internet connection to call the Twilio's API? What if you don't turn on your laptop? Ok, so we should deploy it on a remote server. It should be low-cost, preferably free. And we don't want to have to manage it (restarting the job for example). To recap, we want a service that is

  • fully-managed
  • low-cost/cost-effective (I really mean free, to be honest)
  • reliable

Google Cloud Functions looks perfect for this.

1. What is Google Cloud Functions?

Google Cloud Functions is a lightweight, event-based, asynchronous compute solution that allows you to create small, single-purpose functions that respond to cloud events without the need to manage a server or a runtime environment.

Essentially what this means is that you write a function that will be called upon an HTTP request or other events happening in Google Cloud (a message being published, a file being uploaded to or deleted from cloud storage, etc.), deploy it to Google Cloud and you relax.

Google Cloud Functions is fully-managed, you don't have to grapple with any management overhead. It's free for the first 2 million invocations per month. Only when you used up the free tier that you are billed for subsequent invocations. In our case, it's clear that we won't be paying anything; unless we screw up the job scheduler. Google Cloud Functions is highly available. If the service stops working because of whatever reason, the problem will be handled without your intervention and the service will come back up.

import os
from icalendar import Calendar
from collections import namedtuple
from datetime import date
from twilio.rest import Client
from google.cloud import storage

# Access cloud storage bucket and download the Calendar
BUCKET_NAME = os.environ.get('CLOUD_STORAGE_BUCKET')
storage_client = storage.Client()
bucket = storage_client.get_bucket(BUCKET_NAME)
blob = bucket.get_blob('bday-reminder/cal.ics')
blob.download_to_filename('/tmp/cal.ics')

birth_day = namedtuple('BD', ['name', 'month', 'day'])


def read_calendar():
    ''' Reads the calendar file and generates
    a tuple with name and birthday's month and
    date for each entry in the calendar file '''
    with open('/tmp/cal.ics', 'rb') as f:
        calics = Calendar.from_ical(f.read())
        for component in calics.walk(name='VEVENT'):
            name = component.get('SUMMARY').replace("'s birthday", "")
            birth_date = component.get('DTSTART').dt
            yield birth_day(name=name,
                            month=birth_date.month,
                            day=birth_date.day)


# Your Account Sid and Auth Token from twilio.com/console
account_sid = ''
auth_token = ''
client = Client(account_sid, auth_token)

def send_sms(msg, from_='+166...', to='+233...'):
    message = client.messages.create(
        body=msg, from_=from_, to=to)
    print(message.sid)


def notify(request):
    ''' HTTP Cloud Function
    Checks if today is the birthday of a friend.
    If yes, send out a message '''
    today = date.today()
    bday_names = ''
    for bday in read_calendar():
        if bday.month == today.month and bday.day == today.day:
            bday_names += '\n{}'.format(bday.name)
    msg = '{}{}'.format(today, bday_names)
    if bday_names:
        send_sms(msg)
    return '<br>'.join(msg.splitlines())

3. Birthday data

As I mentioned it earlier, how to export your friends' birthdays data from Facebook and export them as a calendar file (cal.ics) is covered here.

I created a Cloud Storage bucket where I uploaded the calendar file. To work with that calendar file I have to download it first. That's what happening at line 13. The file is downloaded inside /tmp because during execution, a function can only write that location of the filesystem. You may have noticed that I am getting the bucket name from an environment variable. When deploying a cloud function, you can specify environment variables and use them in your code.

4. Let's deploy the app to Google Cloud Functions

Let's deploy our birthday app to Google Cloud Functions using Cloud Console. We will first create a new project. Then we create a cloud storage bucket and upload the calendar file to that bucket. Next, we create a cloud function and subsequently create a job that will call the function on a recurring schedule. I assume that you have a Google Cloud account

a. Navigate to Google Cloud Console and click CREATE to create a new project

b. Enter the project's name and click CREATE

c. Click on Select a project on the app bar and select the project you just created

d. Click on the menu button (amburger icon) and click on Storage to Create a bucket. Then click Create bucket

e. Enter a name for the bucket and click CREATE. Bucket names have to be globally unique

Take note of the bucket name, you will need it in the next steps.

f. Upload the calendar file to the bucket

g. Move the calendar file inside of a folder that can be created by clicking on Create folder. Name the folder bday-reminder

h. Click on the menu button and click on Cloud Functions. Then click Create function

If the Cloud Functions API is not enabled, instead of a button Create function, you would see Enable API. Click on it to enable Cloud Functions and navigate back to the Cloud Functions console to finally create a function.

i. Fill out the form with the relevant information and click Create to deploy your function

  • Function name: bday-reminder
  • Memory allocated, the amount of memory to allocate to the function: 128MB
  • Trigger: HTTP
  • Source code, where to find the source code: Inline editor
  • Runtime, the runtime to use to execute the code: Python 3.7

Copy your code and paste it in the editor as shown below. We could use a different source, we could upload the code to cloud storage and specify the storage bucket, we could zip the code files and just upload it or specify a source repository.

Specify the function to execute. It's the name of the function in the code that will be executed when the Google Cloud Function is triggered: notify

Then click on the link Environment variables, networking, timeouts and more to configure other Cloud Function options as shown below.

The option we are most interested in is Environment variables. Click Add variable to create an environment variable that will store the name of the storage bucket we created earlier as shown below. Finally click Create to deploy your cloud function.

After your function is created, you will see a list of functions in the Cloud Functions console including the one you just created. Click on your newly deployed function to see the dashboard.

When your function was created, a new URL was also created. You can use that URL to trigger your function. Take note of the URL, you will need it.

5. Let's create a job using Google Cloud Scheduler

Now that your code is deployed to Cloud Functions is deployed and ready to be invoked, you are going to create a job to hit the function's URL every day at a specific time to receive the birthday notifation SMS. Click on the menu, then on Cloud Scheduler. You will see the page below, click on CREATE JOB

5.1. Specify a region for your job. You can go with the default

5.2. Fill out the form with the relevant information

  • Name: bday-reminder-job
  • Frequency, the schedule for running the job: 0 17 * * * (Every day at 17:00). Cron expressions can get confusing sometimes, during those times, crontab.guru can help you.
  • Target: HTTP
  • URL: paste your function's URL here.

Finally click Create.

You can set a schedule for your job to be run right away to make sure everything is set up correctly. Click on RUN NOW to test whether your job will run succesfully when its scheduled time time comes.

Check the Result column to see if the job succeeded. If it failed click on View in the Logs column to diagnose the error.
